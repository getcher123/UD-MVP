Контекст:
- мы распознаём страницы PDF-презентаций по объектам недвижимости;
- результат предназначен для аналитиков и брокеров девелопера, которым нужен полный фактический текст без домыслов.

Задача:
1) Внимательно прочитай изображение страницы, двигаясь слева направо и сверху вниз. Каждый визуальный блок (заголовок, подпись, абзац, элемент списка) описывай отдельной записью.
2) Сначала сформируй массив `raw_lines`: перечисли **все** строковые фрагменты в порядке чтения. Если часть текста нечитаема, оставь её как `???`, но всё равно включи в массив.
3) Ничего не выдумывай: если строку нельзя классифицировать, перенеси её как есть в `unknown` (или как `???`, если совсем неразборчиво). 
4) Для каждого объекта массива `blocks` добавь поле `status`, где запиши конкретную оценку качества распознавания; что именно прошло плохо (например: “есть ???”, “часть строк размыта”, “таблица обрезана справа”). Не используй одну и ту же формулировку для всех блоков; если блок без проблем, напиши “полностью читабельно”. Если в `raw_lines` для блока есть `???` или потеряны строки, статус обязан описать, что пропущено.
5) Если текстовый блок представляет строку таблицы, выведи её в формате Markdown-таблицы: ячейки в строке разделяй `|`, например `| NTC | 46 | 556 | 496 | 70,000 | 0 | под отделку | 1,892,200 | сегодня |`.

Чек-лист перед ответом:
1) Убедись, что `raw_lines` покрывает каждую строку страницы без пропусков и следует порядку чтения.
2) Проверь, что каждый элемент из `raw_lines` либо использован в структуре, либо добавлен в `unknown`.
3) Подтверди, что нигде нет выдуманного текста и орфография соответствует оригиналу. В каждом объекте массива `blocks` добавь поле `status`, где кратко опиши качество распознавания блока (например, "полностью читабельно", "частично размыто", "добавлены ???"). 

Формат ответа:
- верни строгий JSON по схеме;
- строки экранируй стандартным образом (`\"`) при необходимости;
- ничего кроме JSON не выводи.
